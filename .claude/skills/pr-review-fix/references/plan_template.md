# 修正計画テンプレート (Phase 3)

Phase 3 ではこのテンプレートを埋めてユーザーに提示する。
**全ての表で先頭列に通し番号 `#` を付け、Phase 6 / Phase 7 でも同じ番号で参照する**
(コメント ID を毎回書かなくて済む一意識別子)。

## ソート規則 (重要)

全テーブルは以下の優先順位で **降順ソート** して表示する。通し番号 `#` はソート後に 1 から振る
(これにより表示順 = 処理順 = 重要度順になる)。

1. **対応方針**: 🔴 > 🟡 > 🟢
2. **収集順** (Phase 1 の取得順、stable sort の tie-breaker)

````markdown
# PR #{PR_NUMBER} 修正計画

## 📋 コメント一覧 (重要度順)
| # | 対応方針 | カテゴリー | ファイル:行 | 内容 | レビュアー |
|---|---------|-----------|-----------|------|-----------|
| 1 | 🔴 | 仕様適合性 | src/auth.js:88 | SQL インジェクションの恐れ | @reviewer3 |
| 2 | 🔴 | アーキテクチャ | src/main.js:15 | 変数名をより具体的に | @reviewer1 |
| 3 | 🟡 | パフォーマンス | src/api.js:42 | キャッシュ機能の追加検討 | @reviewer2 |
| 4 | 🟡 | 可読性・保守性 | src/util.js:10 | 早期 return で読みやすく | @reviewer2 |

## 🎯 修正方針
### 必須修正項目 🔴
- [ ] #1: <具体的な修正内容>
- [ ] #2: <具体的な修正内容>

### 推奨修正項目 🟡
- [ ] #3: <具体的な修正内容>
- [ ] #4: <具体的な修正内容>

## 📅 実装順序
1. 通し番号順 (= 重要度順) に修正
2. テスト・lint・typecheck 実行
3. push → Phase 7 で返信 & resolve

## ⚠️ 注意事項
- 🟢 質問・議論項目は計画対象外
- 修正により既存機能に影響がないか確認
- 依存関係の変更がある場合は慎重に検討
````

## Phase 4 セルフチェックリスト

計画提示前に **内部で** 以下を満たしているか確認する。80% 以上満たすまで Phase 5 に進まない。
**このチェックリストの実行結果 (どの項目が満たされた／満たされていない等) はユーザーに出力しない**
(内部判定専用。満たせていない項目は計画を作り直して解消するだけでよい)。

- [ ] PR 上の **未解決 (unresolved)** 全コメントを反映している (resolve 済みは除外で OK)
- [ ] 全コメントが 🔴 / 🟡 / 🟢 のいずれかに分類されている
- [ ] 全コメントにカテゴリーが付与されている
- [ ] 上記ソート規則に従って **対応方針の降順** にテーブルが並んでいる
- [ ] 通し番号 `#` がソート後に 1 から振られている
- [ ] 修正方針で通し番号を参照している
- [ ] 各コメントの内容が分析されている
- [ ] 具体的かつ明確な修正方針が書かれている
