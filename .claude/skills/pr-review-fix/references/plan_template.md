# 修正計画テンプレート (Phase 3)

Phase 3 ではこのテンプレートを埋めてユーザーに提示する。
**全ての表で先頭列に通し番号 `#` を付け、Phase 6 / Phase 7 でも同じ番号で参照する**
(コメント ID を毎回書かなくて済む一意識別子)。

````markdown
# PR #{PR_NUMBER} 修正計画

## 📋 コメント一覧
| # | 分類 | カテゴリー | ファイル:行 | 内容 | レビュアー |
|---|------|-----------|-----------|------|-----------|
| 1 | 🔴 | アーキテクチャ | src/main.js:15 | 変数名をより具体的に | @reviewer1 |
| 2 | 🟡 | パフォーマンス | src/api.js:42 | キャッシュ機能の追加検討 | @reviewer2 |

## 🎯 修正方針
### 必須修正項目 🔴
- [ ] #1: <具体的な修正内容>

### 推奨修正項目 🟡
- [ ] #2: <具体的な修正内容>

## 📅 実装順序
1. 必須修正項目の実装
2. 推奨修正項目の実装
3. テスト・lint・typecheck 実行
4. push → Phase 7 で返信 & resolve

## ⚠️ 注意事項
- 🟢 質問・議論項目は計画対象外
- 修正により既存機能に影響がないか確認
- 依存関係の変更がある場合は慎重に検討
````

## Phase 4 セルフチェックリスト

計画提示前に以下を満たしているか確認する。80% 以上満たすまで Phase 5 に進まない。

- [ ] PR 上の **未解決 (unresolved)** 全コメントを反映している (resolve 済みは除外で OK)
- [ ] 全コメントが 🔴 / 🟡 / 🟢 のいずれかに分類されている
- [ ] 通し番号 `#` が先頭列に振られている
- [ ] 修正方針で通し番号を参照している
- [ ] 各コメントの内容が分析されている
- [ ] 具体的かつ明確な修正方針が書かれている
