# Commit (dispatcher)

このスキルは **ディスパッチャ**。メインエージェントは `git commit` を自分で実行せず、
必ず **haiku の軽量サブエージェント** に委譲する。実際のコミットポリシーは
[`policy.md`](policy.md) にあり、サブエージェントがそれを読んで実行する。

## メインエージェントの手順

1. ユーザーの依頼・文脈から「何をコミットするか」の指示を把握する (特定ファイルのみ等)。
   指示が無ければ「未コミット変更全体を意味ごとに分割してコミット」とみなす。
2. **Agent ツール** を以下で呼び出す:
   - `subagent_type`: `general-purpose`
   - `model`: `haiku`
   - `prompt`: 下記テンプレート
3. サブエージェントの報告 (作成コミットの一覧) をそのままユーザーに伝える。
   メイン側で `git status` や `git log` を再確認する必要はない。

委譲先で判断に迷うケース (どの変更を含めるか曖昧、分割粒度が不明等) が出た場合、
サブエージェントは**コミットせず状況を報告して終了**する。メインはその報告を受けて
ユーザーに確認してから、必要なら指示を補って再度サブエージェントを呼ぶ。

## サブエージェントへ渡す prompt テンプレート

```
あなたは git commit 専任です。次の手順書を読み、そこに書かれた規約に厳密に従って
現在のリポジトリの変更をコミットしてください。

手順書: ~/.claude/skills/commit/policy.md

追加の文脈・指示 (メインから):
{ユーザーの依頼やコミット対象の指定をここに転記。無ければ「特になし」}

ルール:
- 手順書の判断基準に従い、推測や確認待ちをせず最後までコミットを完了させること。
- ただし「どの変更を含めるか」「分割粒度」が本質的に曖昧でユーザー判断が要る場合のみ、
  コミットせずに状況を簡潔に報告して終了すること。
- 完了したら作成した各コミットを `git log --oneline -<件数>` の形で報告すること。
```
