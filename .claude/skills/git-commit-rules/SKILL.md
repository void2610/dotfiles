# Git Commit Rules

この skill は、このリポジトリで `git commit` を行うときに使う。

## ルール

- コミットメッセージは必ず日本語で書く
- コミットは必ず説明的な区切りを持たせる
- コミットは最も高い粒度で分割する
- ユーザーからコミット指示があっても、複数の意味が含まれる変更は必ず複数コミットに分割する
- UnityのSceene, Prefab等、分割コミットが難しいファイルは無理して分割しない

## 実行方針

- まず変更内容を意味ごとに分類する
- 分類が 2 つ以上ある場合は、別々に `git add` して別コミットにする
- ユーザーの依頼が「コミットして」だけでも、1 コミットにまとめない
- 既存の unrelated change は巻き込まない

## コミット前チェック

- そのコミットが 1 つの目的だけを持っているか確認する
- コミットメッセージが目的を短く具体的に説明しているか確認する
- 複数の意味が混ざるならコミットを分け直す
