# 発想技法リファレンス（Ideation Techniques）

アイデア生成時に多様性を確保するため、以下の発想技法を意識的にローテーションする。
各世代で最低3つの異なる技法を使用すること。

---

## 技法一覧

### 1. 異種交配（Cross-Pollination）
異なるジャンルの要素を強制的に組み合わせる。

**手順**: ジャンルA × ジャンルB × 制約C
**例**: ローグライク × 料理 × 音ゲー → リズムに合わせて食材を切るローグライクダンジョン

組み合わせ元として使えるジャンル・要素プール:
- ゲームジャンル: ローグライク, タワーディフェンス, パズル, リズムゲー, シミュレーション, ビジュアルノベル, メトロイドヴァニア, オートバトラー, デッキビルド, サンドボックス, 脱出ゲーム, クリッカー/放置, レーシング, 格闘, SHMUP
- 非ゲーム領域: 園芸, 天文学, 料理, 裁縫, 経理, 言語学, 建築, 医療, 音楽制作, 郵便, 考古学, 気象, 株式, 生態学, 図書館学

### 2. 逆転の発想（Inversion）
既存ゲームの前提を反転させる。

**手順**: 「通常○○するゲームを、△△しないゲームにしたら？」
**例**:
- 敵を倒すゲーム → 敵を育てるゲーム
- スコアを上げるゲーム → スコアを0にするゲーム
- クリアを目指すゲーム → 永遠に終わらないことを目指すゲーム
- プレイヤーが操作するゲーム → プレイヤーがルールを書くゲーム

### 3. 感覚のゲーム化（Sensation Mapping）
日常の感覚や体験をゲームメカニクスに変換する。

**手順**: 感覚を選ぶ → その感覚の構成要素を分解 → メカニクスに対応づける
**例**:
- 「眠気」→ 視界がぼやける + 反応が遅れる + 意識を保つ努力 → 意識を保ち続けるサバイバルゲーム
- 「満員電車」→ 密度管理 + パーソナルスペース + 出口争奪 → テリトリー確保パズル

### 4. 制約駆動（Constraint-Driven）
極端な制約から出発する。

**手順**: 制約を1つ選ぶ → その制約下で成立するゲームを考える
**制約の例**:
- ボタン1つしか使えない
- プレイ時間が10秒
- 画面に1色しか使えない
- 音だけで遊ぶ（画面なし）
- テキスト入力だけで遊ぶ
- 同時に100人が1つの画面を共有
- セーブできない
- 負けた方が楽しい

### 5. メカニクス抽出（Mechanic Extraction）
非ゲームのシステムからメカニクスを抽出する。

**手順**: 現実のシステムを観察 → ルールを抽出 → ゲーム化
**例**:
- 株式市場 → 需給バランス、投機、バブル崩壊 → 相場操縦ゲーム
- 免疫システム → パターン認識、記憶、過剰反応 → 体内防衛ゲーム
- 生態系 → 食物連鎖、環境適応、共生 → 進化シミュレーション
- 噂話 → 伝播、変異、信頼度 → 情報操作ゲーム

### 6. 感情ターゲット（Emotion Targeting）
先に「与えたい感情」を決め、そこから逆算する。

**手順**: 感情を選ぶ → その感情を最も強く生むシチュエーションを考える → ゲーム化
**感情の例**:
- 「バレないかハラハラ」→ 潜入/偽装ゲーム
- 「意外なつながりに気づくアハ体験」→ 隠れた関連性を見つけるパズル
- 「取り返しのつかないことをした罪悪感」→ 不可逆な選択のゲーム
- 「見知らぬ人との無言の共感」→ 非同期協力ゲーム
- 「じわじわ完成していく満足感」→ コレクション/構築ゲーム

### 7. スケール変換（Scale Shift）
既知の概念のスケールを極端に変える。

**手順**: 概念を選ぶ → 10倍/100倍/1000倍にする or 1/10, 1/100にする
**例**:
- テトリス × 1000倍 → 都市サイズのブロックを積む都市計画ゲーム
- 戦争 × 1/1000 → 蟻同士の縄張り争いシミュレーション
- RPGの1ダンジョン × 1000倍の深さ → 無限に深い穴を掘り続けるゲーム

### 8. 動詞×名詞・名詞先決め（Noun-First Verb×Noun）
動詞×名詞の強化版。何百回も回すと「どんな動詞と組み合わせても面白くなる名詞」が存在することに気づく。そういったS tier名詞を先に決めてから動詞を当てる。

**S tier名詞の例**: 「すべて」「UI」「ルール」

**手順**: 強い名詞を固定 → 動詞をぶつける
**例**:
- 「巻き込む × すべて」→ 塊魂
- 「振り回す × UI」→ UIを武器として振り回すゲーム
- 「押す × ルール」→ Baba Is You

**強い名詞の傾向**: 現実世界では扱えないもの、メタ的な概念（ルール、UI、画面、時間、ゲームそのもの等）を名詞にすると面白くなりやすい。

### 11. ジャンルの固定概念破壊（Genre Assumption Breaking）
既存ジャンルの「暗黙の前提」を列挙し、それを意図的に変更・反転する。

**手順**:
1. 対象ジャンルを選ぶ
2. そのジャンルで「当たり前」とされている要素を列挙する
3. 各要素を別のものに置き換える

**例（プラットフォーマー）**:
- リアルタイムアクション → ターン制にする
- サイドビュー → トップダウンにしてZ軸ジャンプを入れる

**例（RPG）**:
- RPGの世界にプラットフォーマーの概念を持ち込む → 「登山勇者」（RPG風味の縦スクロール登山ゲーム）

異なるジャンルの前提を別ジャンルに持ち込むのが特に強力。

### 9. 舞台を特定のツール/環境にする（Tool-as-Stage）
ゲームの舞台をキーボード、デスクトップ、ブラウザ、ファイルマネージャなど、プレイヤーが普段使っているツールや環境そのものにする。

**手順**: 身の回りのツール/UIを1つ選ぶ → その環境固有の要素（キー配列、ウィンドウ、アイコン等）をギミック化する
**例**:
- キーボード上で繰り広げられるリズムアクション
- デスクトップ上を冒険するプラットフォーマー
- ブラウザのタブを行き来して進むアドベンチャー
- ターミナル(CLI)が舞台のローグライク

ツール固有のギミックが次々と連想されやすいのが強み。0→1というよりは1→10の技法として使えるが、舞台選びの時点で独自性が確保されるため非常に有効。

---

## 技法のローテーション規則

各世代で、以下のように技法を割り当てる：

- **新規個体（Immigration）**: 毎回異なる技法を使う。前世代で使った技法は避ける。
- **突然変異（Mutation）**: 元アイデアの技法とは異なる技法で変異させる。
- **交配（Crossover）**: 技法の指定なし。2つの親アイデアの要素結合に集中する。

### 技法の記録

各アイデアの「出自」欄に使用した技法も記録する：

```
**出自**: 新規（技法: 逆転の発想 - 「倒す→育てる」の反転）
```

これにより、どの技法が良いアイデアを生みやすいかのメタ分析が可能になる。
