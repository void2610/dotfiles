# 評価基準ルーブリック（Evaluation Criteria Rubric）

アイデア評価時に必ず参照すること。各評価軸について、スコア帯ごとの基準を定義する。

---

## 1. 独自性（Originality） - 25点満点

「これは他にない」と言えるか。

| スコア帯 | 基準 |
|----------|------|
| 21-25 | 既存ゲームで類似作品が思い当たらない。メカニクスの組み合わせか、根本的な遊びの構造が新しい。「なぜ今まで誰も作らなかったのか」と思えるレベル。 |
| 16-20 | 既存の要素の組み合わせだが、その組み合わせ自体が新鮮。「○○と△△を混ぜた感じ」と言えるが、混ぜ方に独自性がある。 |
| 11-15 | ジャンルとしては既存だが、1つの要素に明確な差別化ポイントがある。「○○だけど、××なところが違う」。 |
| 6-10  | 既存ゲームのバリエーション。テーマや見た目は違うが、遊びの構造は既存と同じ。 |
| 1-5   | ほぼクローン。既存タイトルを具体的に3つ以上挙げられ、いずれとも大差ない。 |

### 独自性チェックリスト

評価時に以下を自問する：

- [ ] このゲームのコアメカニクスを持つ既存ゲームを3つ挙げられるか？→ 挙げられたら15点以下
- [ ] 「○○みたいなゲーム」の一言で説明が終わってしまわないか？
- [ ] プレイヤーが「こういう体験は初めてだ」と感じるポイントがあるか？
- [ ] メカニクスの新しさか、メカニクスの組み合わせの新しさか、テーマの新しさか？
  - メカニクス自体が新しい → 高得点
  - 組み合わせが新しい → 中〜高得点
  - テーマだけ新しい → 低得点

---

## 2. 面白さの核（Fun Core） - 25点満点

コアループが本質的に楽しいか。テーマやガワを取り除いてもメカニクスだけで面白いか。

| スコア帯 | 基準 |
|----------|------|
| 21-25 | コアループ自体に中毒性がある。「もう1回」が自然に発生する構造。プレイヤーのスキルが上がる実感と、それに伴う新たなチャレンジが内在している。 |
| 16-20 | コアループは楽しいが、持続性に若干の不安がある。最初の30分は確実に楽しいが、3時間後は不明。 |
| 11-15 | アイデアとしては面白いが、実際にプレイすると単調になりそう。変化や深みが不足。 |
| 6-10  | コンセプトは理解できるが、「それ本当に楽しい？」と疑問が残る。頭では面白そうだが体験として微妙。 |
| 1-5   | コアループが不明確、または明らかに退屈。 |

### 面白さチェックリスト

- [ ] 30秒で面白さを体験できるか？（即時の面白さ）
- [ ] プレイヤーの上達を実感できる仕組みがあるか？
- [ ] 「もう1回やりたい」の動機は何か？明確に言語化できるか？
- [ ] テーマを変えても面白いか？（仮にゾンビをカピバラに変えても）
- [ ] ランダム性と実力のバランスは取れているか？

---

## 3. 伝わりやすさ（Communicability） - 15点満点

マーケティング観点。スクショ、1文の説明、短い動画で魅力が伝わるか。

| スコア帯 | 基準 |
|----------|------|
| 13-15 | スクショ1枚またはGIF1つで「やりたい」と思わせられる。タイトルだけでコンセプトが伝わる。 SNSで自然にシェアされる要素がある。 |
| 10-12 | 30秒の動画を見れば魅力が伝わる。説明文2-3行で「なるほど面白そう」となる。 |
| 7-9   | コンセプトの説明に1分以上かかる。面白さは理解できるが、「見てすぐわかる」ではない。 |
| 4-6   | 実際にプレイしないと面白さが伝わらない。マーケティングが困難。 |
| 1-3   | 説明を聞いても面白さがピンとこない。抽象的すぎる。 |

### 伝わりやすさチェックリスト

- [ ] Twitter/Xに1ツイートで魅力を書けるか？
- [ ] スクリーンショットに「何が起きているか」が映るか？
- [ ] ゲーム名だけでジャンルや雰囲気が想像できるか？
- [ ] 友達に「こういうゲームだよ」と3秒で説明できるか？
- [ ] unityroomやSteamのサムネイルで他と差別化できるか？

## 総合評価の注意事項

### 評価のメタ認知

世代が進むにつれて「評価のインフレ」が起きやすい。以下を意識する：

- 第1世代のスコア分布を基準として記憶しておく
- 各世代の平均スコアが前世代より10点以上上がっていたら、基準がブレている可能性がある
- 迷ったら独自性チェックリストに立ち返る

---

## 評価プロセスのフロー

1. 全アイデアを一読する
2. 各アイデアについて5軸のスコアをつける（チェックリストを参照）
3. スコアを一覧にして相対的な違和感がないか確認する
4. 違和感があれば調整（根拠も更新）
5. 特別ルールの適用（キラー、ペナルティ）
6. 最終スコアを確定し、記録する
